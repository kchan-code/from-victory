## Workflow protocols (carry-forward, unchanged)

1. **Never edit `main` directly.** Branch first.
2. **Conventional commits.** `feat:`, `fix:`, `chore:`, `test:`, `docs:`, `build:`.
3. **PR for every change.** Merge through the GitHub UI; never locally.
4. **CI must pass.** Build, typecheck, lint, tests, plus Vercel preview deploy.
5. **Hotfixes** get `hotfix/<name>` branch prefix.
6. **Manual privacy review** before merging any PR touching `apps/web/**`, `supabase/**`, `packages/content/**` (currently empty), `.claude/agents/**`, `CLAUDE.md`, or `docs/brand.md`. Invoke `kids-privacy-officer` and post the verdict on the PR.

KC's operational preferences:
- **Propose-then-execute** for non-trivial change.
- **Single-purpose PRs.**
- **Engineering-clean over pragmatic shortcuts.** No `--no-verify`, no `--admin`, no permission widening.
- **Push back once on scope creep** (product-strategist has veto). Reaffirm and proceed.
- **No "take a beat" suggestions** — set own cadence.
