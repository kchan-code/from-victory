## Things that took longer than they should have

Recorded so the next session doesn't repeat them:

1. **Days 6-10 first draft was tonally wrong.** The arc block theme "Resetting after mistakes" structurally trapped every day into opening with a problem instead of a truth. KC's reaction: "really negative." Burned ~1.5 hours of trio work. Reframed as "Receiving grace" — much better fit. Lesson: pressure-test arc-block themes for tonal arc before drafting 5 days.
2. **Sign in nav got added, reverted, re-added.** PR #40 added a Sign in link hidden below `sm:` — KC reverted it (probably because mobile users couldn't see it). PR #43 re-added it properly. Lesson: when adding nav entries, default to visible-on-mobile unless there's a strong reason not to.
3. **18+ direct-signup almost shipped before product-strategist veto.** KC said go on A+A+A; I almost built it. Product-strategist's veto reasoning was strong (scope creep, orphan-athlete data debt, Stripe migration trap). Pivoted to the admin tool — much cleaner. Lesson: when a CLAUDE.md "Open Items" entry says explicitly "post-MVP," that label is load-bearing. Pulling it forward needs justification beyond "beta tester unblock."
4. **Reset-password email pointed to localhost** because Supabase project Site URL was still the default `http://localhost:3000`. I'd asked KC to add the redirect URL to the allowlist (correct), but never the Site URL itself. Lesson: when shipping anything that sends Supabase-templated emails, check BOTH the redirect-URL allowlist AND the Site URL. Site URL is what email templates use as the base.
5. **Pregame flow had a real bug at PhaseIndicator** — kids-privacy-officer caught it during privacy review (visualization phase label rendering empty because `step.kind.startsWith("l")` matched none of the actual kinds). Fixed before push. Lesson: privacy review can catch real bugs, not just privacy concerns.
