## Workflow protocols to honor

Codified in CLAUDE.md "Workflow Rules" section:

1. **Never edit main directly.** Every task → `git checkout -b <type>/<name>` first.
2. **Small logical commits.** Conventional prefixes: `feat:`, `fix:`, `chore:`, `test:`, `docs:`.
3. **PR for every change.** Merge through GitHub UI, never locally.
4. **CI must pass:** build, typecheck, tests, lint. (Privacy review is NOT in CI anymore — see rule 6.)
5. **Hotfixes** use `hotfix/<name>` prefix, same flow.
6. **Manual privacy review** before merging any PR touching `apps/web/**`, `supabase/**`, `packages/content/**`, `.claude/agents/**`, `CLAUDE.md`, or `docs/brand.md`. Invoke kids-privacy-officer subagent in your session, post review as PR comment. (Auto-CI gate will return when privacy code actually lands.)

Operational pattern KC prefers:

- **Propose-then-execute.** Before a non-trivial change, lay out the concrete file plan and wait for KC's OK. Don't just start writing.
- **Workflow per PR:** branch → propose → wait for OK → write → build → lint → show status + diff stat → commit → push → PR → **never merge**.
- **Single-purpose PRs.** Don't bundle unrelated changes.
- **Engineering-clean over pragmatic shortcuts.** No `--no-verify`, no `--admin` overrides (rulesets block them anyway), no permission widening. Fix root causes.
- **Push back on scope creep.** product-strategist has veto. If KC chooses an option both you and the agent recommended against, push back once, plainly. Then if reaffirmed, proceed.
