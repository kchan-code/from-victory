## Architecture decisions worth carrying forward

### Build-time TTS over runtime

Decided early in the session. Real audio playback was the chosen
production model (vs. on-screen-only personalization or stitched
segments). KC picked **build-time generation, manual local run**:
generator runs on developer's machine with OPENAI_API_KEY in
`apps/web/.env.local`, MP3s commit to the repo, Vercel just serves
static. **Zero athlete data ever touches OpenAI.** Sidesteps any
third-party-processor question about minor-typed free text.

When the 30-script content lands, expect ~$0.05 total to generate all
of them once.

### Pause durations are data, not estimates

Each script segment is typed as `{ type: "speech" | "silence", ... }`.
Silences are explicit `durationSec` values; generator creates a silent
MP3 of that length and concatenates. Sidecar JSON timeline records
absolute phase start times based on **measured** post-TTS durations
via ffprobe — runtime sphere sync stays correct even when TTS clips
run shorter/longer than estimates.

### Runtime sync via rAF, not timeupdate

`audio.timeupdate` fires every ~250ms in most browsers (slower on iOS).
That gave visibly choppy sphere expansion/contraction at first. Fixed
in PR #49: `useBreathAudio` runs a `requestAnimationFrame` loop while
`isPlaying`, reading `audio.currentTime` at 60fps. Important pattern
to reuse for any future audio-driven visual sync work.

### Controlled vs. standalone BreathingSphere

The sphere now has two render modes. **Controlled mode** (parent
supplies `controlled={{ phase, t, round, isPlaying }}` from
`useBreathAudio`) is what the production breath screen uses. **Standalone
mode** (sphere owns an internal rAF timer; tap-to-start) is the fallback
when audio/JSON 404. Both code paths shipped. Falls through gracefully.

### Per-segment speed override

`Segment.speed?: number` added in PR #50. Lets the intro run at
conversational pace (1.0–1.1) while breath cues stay meditative (0.95).
**Pattern to reuse for the 5-min session** when it lands — settle
segments will want 0.88, visualization can run 1.0, prayer goes back to
0.88, etc.

### How to regenerate audio (operator runbook)

From `apps/web/`:

```sh
# one-time setup
brew install ffmpeg
echo 'OPENAI_API_KEY=sk-...' >> .env.local

# generate
npm run audio:generate                                # all scripts in SCRIPTS[]
npm run audio:generate -- --dry-run                   # validate + estimate cost
npm run audio:generate -- --slug breath-threshold     # one slug
npm run audio:generate -- --keep-segments             # leave per-segment files for inspection

# commit
git add apps/web/public/audio/pregame/
```

Costs ~$0.0001 per breath script regeneration. ~$0.05 once the 30
guided-session scripts are written.

### Key gotchas worth remembering

- **macOS `sips` rasterizes SVG to PNG** — no need to install ImageMagick
  or librsvg. Used for the home-screen icons.
- **`gpt-4o-mini-tts` doesn't support SSML** — pauses have to come from
  silence segments, not `<break>` tags.
- **iOS `apple-touch-icon` requires PNG**, not SVG. SVG won't render on
  the home screen.
- **`audio.timeupdate` is too slow for animation sync** — use rAF reading
  `audio.currentTime` instead. See `useBreathAudio.ts`.
- **`node --experimental-strip-types` requires explicit `.ts` extensions**
  in imports. `tsconfig.json` has `allowImportingTsExtensions: true` to
  keep tsc happy with the imports inside `scripts/`.
- **The "Receive" subtitle on the Breath screen is on-screen-only.** It's
  not in the audio for round 2 (intentionally trimmed). Don't re-add
  unless KC asks.
